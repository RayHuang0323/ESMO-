## Sprint 25 完成（Unified Match Rewards & Player Progress）

Audit 發現的現況比預期嚴重：**MOBA 打一百場，錢 / 粉絲 / 選手 XP 全部不動**；
CS 只有錢和粉絲會動；選手 XP **根本不存在**（players[] 有 lv 但無 xp 欄位，
lv 是 Legacy 靜態種子）；天賦點閉環從來不存在。

- 新增回寫交易層 `MatchProgressTransaction.v1`（**不合併**兩個 Result 契約、
  **不合併** MOBA history 與 csHistory）。
- 各遊戲 Reward Adapter → 單一 Progress Service `applyMatchProgress`（唯一發獎點）
  → Receipt → Result Screen 只顯示、不重算。
- 冪等：`transactionId = mode:matchId:progress-v1` + `processedMatchTransactions` 帳本
  ⇒ StrictMode 雙掛載 / 重整 / 返回再進 Result 都不重複發獎。
- 結算移到**比賽完成邊界**（MOBA = 引擎終局、CS = onFinish），
  不再掛在 Result Screen 掛載 ⇒ 跳過 Result 也不會漏發獎。
- 選手 XP / 等級 / 天賦點閉環成立（`players[].xp` 為累積總 XP，lv 由 xp 導出）。
- persistence migration（schemaVersion 2）：舊存檔安全升級，等級不倒退、不清資料。
- 修正真實缺陷：MOBA 輔助因 playerRating 未正規化而 XP 只有隊均 70%
  → 改以 participation 為主後回到 91%。
- 驗證：check_progress25 34/34、check_cs23 28/28、check_moba_tactic24 27/27、
  regress/regress2、build 全綠（**全部檢查 exit code**）。

設計文件：`docs/design/賽後結算與選手成長系統.md`
