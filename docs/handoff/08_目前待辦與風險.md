## Sprint 26 殘留技術債（對戰體驗）

1. **重播僅存 session 記憶體最近一場**——localStorage 配額不適合 frames
   （345KB/場），重整即失、Result 顯示「無法重播」。要多場持久化 →
   IndexedDB + delta 壓縮（見 docs/design/MOBA重播系統.md 未來節）。
2. **重播是 2D 俯視**——3D 重播需要播放管線與 live 管線（useGameStore/
   useBattleFeed）完全隔離，否則會誤觸發終局結算；列未來項。
3. **手機各檔位未經真機實測**（本環境無瀏覽器）——B 修復只有靜態防線
   驗證（sticky footer / 寬度防護 / grid min()），320/360/390/430/768/1366/1920
   需人工點一輪。
4. **「最近使用英雄」未實作**——profileStore 沒有真實 lastHero 資料來源；
   若產品要在選手卡顯示，需先在賽後結算寫入真實的最近英雄紀錄（標示「最近使用」）。

## Sprint 25 殘留技術債（賽後結算與選手成長）

1. **MOBA matchId 是內容雜湊** —— `BattleResult.v2` 沒有 matchId 也沒有時間戳，
   契約凍結不能加，因此 `mobaMatchId()` 以比賽內容做 FNV-1a 決定性雜湊。
   影響：兩場「完全相同」的比賽（勝方/時長/比分/10 人 KDA・金錢・傷害全等）
   會被視為同一場 → 第二場不發獎。機率極低，且比 seasonStore 現有的 4 欄位
   `resultKey` **更安全**。根治方式：讓 BattleResult 帶 matchId（**需 Ray 核准改契約**）。
2. **CS 團隊獎金略高於 MOBA**（fixture：43 萬 vs 33 萬，1.3×）。
   因 CS 的 marginF 用回合差（16:9 → 7/8），MOBA 固定 3/8。
   在容忍範圍內，Legacy 公式凍結，**本 Sprint 未重新平衡**，列為觀察項。
3. **聲望（reputation）永遠 0** —— 無經 Legacy 驗證的公式，不編造。
4. **天賦點只進不出** —— 見上。
5. **team.lv/xp 刻度仍未統一**（「萬 XP」展示刻度，與 xpGain 50/20 不符）。
   S25 刻意不碰：等級閉環做在「選手」層（交易契約的 teamRewards 本來就只有
   money / fans / reputation，沒有 xp）。
6. **老將升級極慢**（lv38→39 需 1950 XP ≈ 25–40 場）。刻意設計，若體感太慢
   需調 `xpRequiredForLevel`。
7. `check_mount09` 仍失效（檢查 Sprint09 時代畫面，非現役）。
   （`check_flow09` / `check_dash10` 的失效 import 路徑已於 S25 修復並復活。）

## Sprint 21 殘留待辦（經營模組）

本 Sprint 恢復了八個經營模組，以下為刻意未做、需後續處理的項目：

1. **賽後金流未回寫財務** —— `matchRecorder` 目前不寫 `profileStore.finance`。
   賽事獎金 / 薪資支出仍是 Store 的種子資料，不會因比賽變動。
   下一步：在 matchRecorder 增加財務回寫（注意：不得在經營層重算戰績）。
2. **贊助週收入未結算** —— `activeSponsor.weeksLeft` 不會自動遞減，
   每週收入也未入帳。需要一個「推進週」的時間軸入口（Legacy 靠 advanceDay 連動）。
3. **經營能力值不影響 Battle** —— 訓練提升的是經營端數值；引擎 loadout 仍用預設
   `HERO_ASSIGN`，Battle Balance 凍結。若要讓選手能力影響勝負，屬 Contract / Balance
   變更，需 Ray 核准。
4. **新秀未綁定英雄** —— `signProspect` 後 `heroId = null`（UI 誠實顯示「未綁定英雄」）。
   要讓新秀能上場，需要一個「指派英雄」入口。
5. **轉會市場 / 合約談判** —— Legacy `NegotiationModule` 領域，未恢復。
6. **天賦 / 商店 / 經營儀表板** —— Legacy Talent / Equip / Dash 模組，未恢復，
   Dashboard 仍為誠實佔位 Modal。
7. **粉絲門檻形同虛設** —— `meta.fans` 種子值 128,000 遠高於 Legacy 贊助最高門檻
   （3,000 粉絲），贊助條件實際只由勝場把關。要恢復 Legacy 手感需重新校準種子值。
