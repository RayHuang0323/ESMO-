# 06 目前主幹架構

## 入口

一般情況：

- `src/main.jsx` 掛載 React App。
- `src/AppShell.jsx` 管理目前主流程。
- `src/GameView.jsx` 負責進入 3D Battle。

## 主要畫面

### Dashboard

- `src/screens/DashboardScreen.jsx`

用途：

- 主選單首頁。
- 經營資訊入口。
- MOBA / CS / 賽事入口。

### MOBA 賽前流程

目前相關檔案：

- `src/screens/moba/RoleSelectScreen.jsx`
- `src/screens/moba/MatchmakingScreen.jsx`
- `src/screens/moba/BanPickScreen.jsx`
- `src/screens/moba/TacticScreen.jsx`
- `src/screens/moba/LoadingScreen.jsx`
- `src/screens/moba/CodexScreen.jsx`
- `src/screens/moba/HeroCodexDetail.jsx`

正確流程應為：

Dashboard  
→ 5 人賽前配置 / RoleSelect 或 Lineup  
→ Matchmaking  
→ BanPick  
→ Tactic  
→ Loading  
→ GameView / Battle  
→ Result  
→ Dashboard

## Hero 資料

唯一英雄資料來源：

- `src/data/heroDatabase.js`

包含：

- 英雄基本資料
- 定位
- 技能名
- 技能描述
- 戰鬥數值
- 成長數值
- 技能 CD / 傷害 / ratio
- 優勢 / 弱點

## Roster

- `src/data/roster.js`

用途：

- 選手 / 隊伍 / 預設英雄。
- MOBA / Battle / Dashboard 部分資料來源。

## Store

主要 Store：

- profileStore：經營資料、隊伍 meta、財務、贊助、收件匣。
- seasonStore：賽季 / 戰績。
- heroProgressStore：英雄成長 / 等級 / 場次。
- battleStore：戰中 snapshot / battle UI 資料。

## Legacy 規格來源

Project Knowledge 中的：

- `EsportsGame.jsx`

用於讀 Legacy UI / UX / 模組 / 英雄圖片 / 舊資料表。

它不是目前主幹。
