# 03 開發規範

## 基本原則

- 小步修改。
- 不大幅重構。
- 不為了方便重寫 UI。
- 不建立第二套資料來源。
- 不破壞已驗證的 Battle Balance。
- 優先可維護、可擴充、可讓 AI 接手。

## 禁止修改項目

除非任務明確允許，禁止修改：

- `src/LogicEngine.js`
- Battle Balance
- BattleResult 結構
- HeroProgress 計算規則
- SeasonStore 統計規則
- battleStore 核心資料流
- Router 主架構

## 禁止新增

- 第二套 Hero Database
- 第二套 Store
- 第二套 BattleResult
- 第二套統計邏輯
- 假資料覆蓋真資料

## Store 新欄位規則

新增 Store 欄位時，必須向下相容 localStorage。

陣列欄位要使用：

```js
Array.isArray(saved.items) ? saved.items : DEFAULT.items
```

UI 讀陣列時，使用：

```js
items ?? []
```

避免黑畫面。

## Hero 資料規則

`src/data/heroDatabase.js` 是唯一英雄資料來源。

不可另建第二套 Hero image database、skill database 或 stats database。

如需圖片，應整合到同一份 heroDatabase 或建立由 heroDatabase 引用的單一資源 map，並清楚說明。

## BattleResult 規則

BattleResult 是唯一戰鬥結果來源。

Result / HeroProgress / Season / Dashboard 不得重新統計 KDA、MVP、勝負、EXP、戰績。

## 驗證要求

每次修改後至少確認：

- esbuild 或 build 通過。
- Contract 檔零改變，除非任務允許。
- 20 seed 回歸不變，若任務不應影響戰鬥。
- UI 流程本機驗證。
- Legacy Diff Checklist。
