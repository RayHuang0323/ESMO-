# 更新日誌 — Sprint 05：Battle Balance

## 修改檔案
- `LogicEngine.js`：AI 行為 + 戰鬥數值（詳下）
- `gameData.js`：`TOWER_HP 1100→2100`、`NEXUS_HP 2600→7200`

## 根因（Node 多 seed 量化，非臆測）
修改前基準（15 seed）：**結束率 0/15、全打滿、平均 0.3 擊殺、撤退鎖死 14/15**。
1. **主堡永不可破（頭號死因）**：nexus 建立時無 `t` 屬性 → 小兵 reach 判定 `lead >= NaN` 恆 false；英雄清路後 `frontTower` 回 null 也不 fallback nexus。實測 10 seed 主堡血全程 2600/2600 滿血。
2. **撤退鎖死（次死因）**：泉水不回血 + 撤退門檻 25% 永久成立 → 全隊擦傷即永久退場（平均 161s），此後零交戰、零擊殺、團戰歸零、龍巴龍單方刷。
3. **對稱僵持**：無優勢差 → 沒有任一路 3 塔清空 → 圍攻/主堡永不觸發。

## AI 行為改動
- **撤退遲滯（hysteresis）**：<25% 進入撤退、回到 60% 才重返；死亡復活重置。解除全隊卡撤退死循環。
- **回血機制**：靠泉水 20%/s 快速回、脫離戰鬥 2%/s 緩回。
- **動態推進**：英雄壓向該路敵方前線塔（塔破 frontTower 前移 → 自然逐塔推進），取代固定 adv 上限 0.7（原本永遠搆不到內塔 t=0.85）。
- **圍攻狀態**：該路清空後英雄湧向敵方主堡（新 `圍攻` 狀態）+ 英雄推塔 fallback 至 nexus。
