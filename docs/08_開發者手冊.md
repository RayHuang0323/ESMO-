# esmo · MOBA 3D（Local Server Pattern）

## 架構

```
src/gameData.js        共用常數/座標工具（地圖、職業、隊色）— 無相依
src/LogicEngine.js      對戰大腦（純邏輯，可搬 Node.js 當多人伺服器）
src/useGameStore.js     zustand：prev / snapshot / subTRef / hud
src/useLocalServer.js   驅動水管：tick 引擎 → pushFrame
src/MobaView3D.jsx      R3F 3D 渲染（Canvas/相機/燈光/Bloom/SSAO）
src/GameView.jsx        整合殼：3D + HUD + 小地圖 + Start 按鈕
src/assets/rift.jpg     彩色 LOL 風格地圖底圖
src/App.jsx / main.jsx  頁面進入點
```

資料流是單向的：`LogicEngine.tick()` 算出新狀態 → `useLocalServer` 把
`snapshot()` 推進 `useGameStore` → `MobaView3D` 的 `useFrame` 只讀 store
做平滑內插渲染,完全不碰邏輯。未來要換成多人 WebSocket,只需把
`useLocalServer.js` 換成監聽 `socket.on("snapshot", pushFrame)`,其他檔案
不用動。

## 本機/Bolt 執行

```bash
npm install
npm run dev
```

開啟瀏覽器看到畫面後,按 **▶ 開始遊戲 / START** 即可觀看 3D 對戰。

## 常見問題

- **SSAO 報錯**:把 `MobaView3D.jsx` 裡 `<EffectComposer enableNormalPass>` 的
  `enableNormalPass` 拿掉,或整段移除 `<SSAO/>`。
- **地圖方向相反**:在 `MobaView3D.jsx` 載入貼圖那行加 `t.flipY = false`。
- **想換真實 LogicEngine**:保持 `snapshot()` 回傳形狀不變(players/towers/
  lanes/fx/feed…),其餘檔案不用改。
