# MOBA 地圖可讀性規範（Sprint 29B3）

> 目標：玩家**看得懂地圖**——不是最終美術。方向參考 Ray 提供的舊地圖平面圖，
> 未使用任何未授權素材、未照抄。全部為程序化繪製/低模/文字標籤。
> 座標唯一來源不變：`gameData.js`（LANES/PITS/CAMPS/BASE/FOUNTAIN/BUSHES）。
> **本 Sprint 未改任何 gameData 座標。**

## 2. 塔與主堡光效（29B3 收斂）

| 狀態 | crystal emissive | 備註 |
|---|---|---|
| 常態 | 塔 0.55／主堡 0.75（29A 1.3/1.9 → 29B2 0.8/1.1 → 29B3 再降） | 主堡 PointLight 5→3.5 |
| 被攻擊 | +1.6 峰值、0.25s 衰減（hp 差分觸發） | 「亮」變成資訊而非常態 |
| 被摧毀 | viewFx 爆點圈（主堡 7／塔 4.5 尺寸） | 之後只剩殘樁 |

Bloom 依畫質分級：low 0.7／medium 0.9／high 1.05（`quality.js bloomIntensity`）。
未新增任何常駐 PointLight（29A 教訓：2 盞上限維持）。

## 3. 標籤實作備註

- 地面字：`makeRiftTexture` canvas `fillText`（一次性繪製，零 runtime 成本）；
  俯視導播角度下輕微斜視，可讀性足夠（MVP 取捨）。
- billboard：`makeLabelSprite` 重用（CanvasTexture sprite，常駐 7 枚：2 pit＋2 泉水＋
  6 營地中 buff/camp 標籤——每枚一張 128×64 canvas，記憶體可忽略）。
- 若 29B4+ 導入正式美術：以本表為「元素必須可辨識」的驗收清單，標籤可隨資產退場。
